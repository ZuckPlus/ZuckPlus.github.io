Week 2: Learning Express.js

This week, I focused on learning Express.js, a popular backend framework for Node.js. Since I already have experience with front-end development, I wanted to explore the backend side of web applications, where I can manage data, handle user authentication, and build APIs.

What is Express.js?

Express.js is a Node JavaScript framework that simplifies working with HTTP requests and responses. Instead of manually setting up servers with raw Node.js, Express provides an easier way to define routes, manage middleware, and structure applications efficiently. I really liked how easy it was to learn this framework. There is tons of documentation and videos explaining how this framework is different and how it is used.

My First Impressions

Getting started with Express was straightforward. With the easy access of knowledge I picked it up quick. I installed it using:

npm install express

Then, I created a simple server using Express. 


const express = require('express');
const app = express();

app.get('/', (req, res) => {
    res.send('Hello, Express!');
});

app.listen(3000, () => {
    console.log('Server is running on port 3000');
});

This was my first hands-on experience setting up a server in Express, and I already see why Express is widely used.
