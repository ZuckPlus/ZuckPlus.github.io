Week 6:Fastify API Security & Low Level C++ Security

Adding onto Fastify API Security

Security is the backbone of any API. This week, I implemented additional measures to prevent common attack vectors such as SQL injection, XSS, CSRF, and request forgery.

  • Input Validation: Enforcing strict schemas for requests.
  • Rate Limiting: Using Fastifys rate-limit plugin to prevent abuse.
  • JWT Hardening: Using stronger encryption and expiring tokens quickly.
  • HTTPS Everywhere: Enforcing TLS and refusing non-secure connections.

Low-Level Security in C++ Loader

The C++ loader must also be secured to prevent tampering and unauthorized access. Here are a few techniques I implemented:

  • Anti-Debugging Techniques: Checking for debuggers and modifying execution flow.
  • Obfuscation: Encrypting critical strings to prevent easy reverse engineering.
  • Memory Protection: Encrypting sensitive data in RAM and clearing it after use.
  • Anti Breakpoint: Checking for break points with a debugger.
  • Good control flow: Making sure that a reverse enginer can't just jump from the auth to the loading function
  • Server side auth: Making sure that all of the authentication process is done on the server and not on the client so it can't be maniuplated.
  • Xor string:Xor string is a string encryperting program that is open source. I have modified it to be much more advanced.
